Comparing random effects between mgcv, nlme, and lme4

Author

Mike O’Brien

https://gist.github.com/mhpob/73cb07f5ef3b6b55f4ef9da8cd0b7b3e

I believe that the model is miscoded in this analysis based on the code provided in the GitHub page and description in this paragraph. The random effects are coded as

s(year, by = !yearblockA, bs = ‘re’) + s(year, by = !yearblockB, bs = ‘re’)+ s(year, by = !yearblockC, bs = ‘re’)

This specifies a random slope by groups of year around an unspecified fixed effect of yearblock. In lme4 or nlme, it lines up with ~(0 + yearblock|year) and an implied, but unparameterized, fixed effect of yearblock:

library(mgcv)

In the BodyWeight data, Rat is nested in Diet with no repeated Rats across Diets

Analogous to Year nested in YearBlock with no repeated Years across YearBlocks

Diet is not coded

data(‘BodyWeight’)

Dummy code levels of Diet as for YearBlock in manuscript

BodyWeight\(Diet1 <- ifelse(BodyWeight\)Diet == 1, 0, 1) BodyWeight\(Diet2 <- ifelse(BodyWeight\)Diet == 2, 0, 1) BodyWeight\(Diet3 <- ifelse(BodyWeight\)Diet == 3, 0, 1)

Random effects structure in the ms

ms_gam <- gam(weight ~ s(Rat, by = ordered(!Diet1), bs = ‘re’) + s(Rat, by = ordered(!Diet2), bs = ‘re’) + s(Rat, by = ordered(!Diet3), bs = ‘re’), data = BodyWeight, method = ‘REML’)

Non-dummy-coded structure

gam_short <- gam(weight ~ s(Rat, by = Diet, bs = ‘re’), data = BodyWeight, method = ‘REML’)

Random slope of Diet, grouped by and uncorrleated across Rat

ms_lme <- lme(weight ~ 1, random = ~ 0 + Diet|Rat, data = BodyWeight)

Similar estimates

gam.vcomp(ms_gam) gam.vcomp(gam_short) VarCorr(ms_lme)

If this is the desire, I suggest including yearblock as a fixed effect as well as the “by” variable in the random effect: yearblock + s(year, by = yearblock, bs = ‘re’) would represent random variation around the effect of yearblock due to year. There is ongoing debate about whether a random effect with less than c. 5 levels can be properly parameterized, and one suggestion is to fit factors with less than 5 levels as fixed effects. This would follow the suggestions outlined in Harrison, X.A., Donaldson, L., Correa-Cano, M.E., Evans, J., Fisher, D.N., Goodwin, C.E., Robinson, B.S., Hodgson, D.J., Inger, R., 2018. A brief introduction to mixed effects modelling and multi-model inference in ecology. PeerJ 6, e4794.

ms_gam2 <- gam(weight ~ Diet + s(Rat, by = Diet, bs = ‘re’), data = BodyWeight, method = ‘REML’)

ms_lme2 <- lme(weight ~ Diet, random = ~ 0 + Diet|Rat, data = BodyWeight)

gam.vcomp(ms_gam2) VarCorr(ms_lme2)

If the desire is to have a random intercept of year and random slope of yearblock by year, s(year, bs = ‘re’) should also be added to parameterize the random intercept. mgcv cannot fit correlated random slope and random intercept (~Diet|Rat or ~1 + Diet|Rat) so they must be modeled as uncorrelated.

ms_gam3 <- gam(weight ~ Diet + s(Rat, bs = ‘re’) + s(Rat, by = Diet, bs = ‘re’), data = BodyWeight, method = ‘REML’)

ms_lme3 <- lme(weight ~ Diet, random = list(~ 1|Rat, ~ 0 + Diet|Rat), data = BodyWeight)

Similar estimates, but starting to drift since model is overparameterized (note wide CIs)

gam.vcomp(ms_gam3) VarCorr(ms_lme3)

If the desire is to have nested random intercepts, then s(year, bs = ‘re’) + s(yearblock, bs = ‘re’) would suffice but might preclude the variance decomposition noted in line 264 and used elsewhere in the manuscript (L373-379; Figures S1, S2). This also creates a random effect with only a few levels, so there is risk that the random effect of Diet (YearBlock) could be poorly parameterized.

nlme doesn’t have a straightforward way to model crossed effects, but it would be

similar to this plus some extra terms to scale the variance. lme4 handles crossed

effects and would be coded similarly to the below

crossed <- lme(weight ~ 1, random = list(~ 1 | Diet, ~1|Rat), data = BodyWeight)

nested <- lme(weight ~ 1, random = ~ 1 | Diet/Rat, data = BodyWeight)

nested_gam <- gam(weight ~ s(Diet, bs = ‘re’) + s(Rat, bs = ‘re’) , data = BodyWeight, method = ‘REML’)

Since no levels of Rat are duplicated across Diet, they are the same

Analogous to no levels of Year duplicated across YearBlock

VarCorr(crossed) VarCorr(nested) gam.vcomp(nested_gam)
